# GitHub Discussions setup

Enable Discussions in your GitHub repo settings, then create these categories:

| Category | Format | Purpose |
|----------|--------|---------|
| 📣 Announcements | Announcement | New tools, major updates — maintainers only |
| 💡 Ideas | Open-ended discussion | Suggest new tools, countries, features |
| 🌍 My country | Open-ended discussion | Country-specific tips and corrections from immigrants |
| 🗣 Translations | Open-ended discussion | Coordinate translation work, discuss terminology |
| ❓ Q&A | Question / Answer | Help using the tools, contribution questions |
| 🙏 Show & Tell | Open-ended discussion | Share how the tools helped you |

## How to enable

1. Go to your repo → Settings → Features → tick "Discussions"
2. Go to the Discussions tab → click "Set up discussions"
3. Create the categories above manually

## Why Discussions instead of Issues

Issues are for bugs and concrete tasks with a clear resolution.
Discussions are for open-ended conversation — country tips, translation coordination, ideas, personal stories.
Keeping them separate makes the Issues tracker actionable and easy to triage.
